# Etapas del Desarrollo de Software

## Introducción

El desarrollo de software es un proceso estructurado que permite crear aplicaciones y sistemas informáticos de manera organizada y eficiente.

---

# 1. Análisis de requisitos

El análisis de requisitos consiste en identificar las necesidades del cliente y las funcionalidades que deberá cumplir el software.

## Requisitos funcionales

Definen las funciones y comportamientos específicos del sistema.

Ejemplo:
- Inicio de sesión
- Registro de usuarios
- Generación de informes

![Requisitos funcionales](https://i.imgur.com/os2DLwi.png)

---

## Requisitos no funcionales

Definen características de calidad del sistema.

Ejemplo:
- Seguridad
- Rendimiento
- Escalabilidad

![Requisitos no funcionales](https://knowgarden.wordpress.com/wp-content/uploads/2014/12/requisitos.jpg?w=640)

---

# 2. Diseño

En esta etapa se planifica la arquitectura y estructura del software antes de programarlo.

Se crean diagramas, modelos de datos e interfaces.

![Diseño](https://i.ytimg.com/vi/ZUSFkY6ZTRQ/maxresdefault.jpg)

---

# 3. Codificación y compilación

Consiste en escribir el código fuente utilizando lenguajes de programación.

La compilación transforma el código en un programa ejecutable.

```python
def saludar():
    print("Hola mundo")

saludar()
```

![Codificación y compilación](https://edteam-media.s3.amazonaws.com/blogs/original/0fd9842c-a1ff-4141-b05f-672fb0811ead.png)

---

# 4. Pruebas

Las pruebas permiten detectar errores y asegurar el correcto funcionamiento del software.

## Pruebas unitarias

Verifican pequeñas partes del programa de forma individual.

## Pruebas de integración

Comprueban el funcionamiento conjunto de diferentes módulos.

![Pruebas](https://edteam-media.s3.amazonaws.com/blogs/original/8442c5cf-38b6-4e18-a5c0-2de515f8794a.png)

---

# 5. Explotación

Es la etapa en la que el software se pone en funcionamiento y es utilizado por los usuarios.

![Explotación](https://img.magnific.com/foto-gratis/concepto-marketing-redes-sociales-marketing-aplicaciones_23-2150063163.jpg)

---

# 6. Mantenimiento

El mantenimiento de aplicaciones tiene como objetivo modificar y actualizar la aplicación después de su entrega para corregir fallos, mejorar el rendimiento, mejorar el diseño, crear interfaz con otros sistemas, implementar mejoras o desarrollar nuevas funcionalidades. Todo este proceso suele estar gestionado en base al control de calidad, la visibilidad del cliente y ciclos ágiles de desarrollo.

Tipos:
- Correctivo
- Preventivo
- Evolutivo

![Mantenimiento](https://t4.ftcdn.net/jpg/03/89/77/85/360_F_389778584_SIoNpPinzODANlY7C5mrfr2VFtYEpWE7.jpg)

# 7. Documentación

Es la información enfocada en la descripción del sistema o producto para quienes se encargan de desarrollarlo, implementarlo y utilizarlo. Por lo que también puedes determinar como una documentación de procesos. Además, esta documentación software incorpora los aspectos de los manuales, como las funciones de ayuda, sus versiones en línea y demás.

Asimismo, permite la comprensión del producto de software, su interfaz, sus capacidades, propiedades principales y demás. Además, la documentación de software será clave para facilitar encontrar de manera rápida una determinada sección.

## Tipos de documentos

- Manual técnico
- Manual de usuario
- Documentación de instalación
- Diagramas UML
- Especificaciones técnicas
